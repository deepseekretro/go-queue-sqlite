# GoQueue — 定时任务示例（cron-dashboard）

纯浏览器 HTML + JS 实现的 GoQueue Cron 定时任务管理界面，无需任何构建工具，直接双击打开即可使用。

## 快速开始

```bash
# 直接用浏览器打开
open goapp/examples/cron-dashboard/index.html

# 或者用任意静态服务器
npx serve goapp/examples/cron-dashboard
python3 -m http.server 3000 --directory goapp/examples/cron-dashboard
```

然后在页面顶部填写服务端地址（默认 `http://localhost:8080`）和 API Key，点击「应用配置」即可。

## 功能说明

### 创建定时任务

填写以下字段后点击「创建定时任务」：

| 字段 | 说明 |
|---|---|
| 任务名称 | 便于识别的名称，不影响执行 |
| 队列名 | 任务投递到哪个队列，默认 `default` |
| Job Type | Worker 侧用于路由的任务类型标识 |
| 执行间隔 | 支持 `10s` / `1m` / `5m` / `1h` / `1d` 等，也可自定义 |
| 优先级 | 1-10，数字越大优先级越高，默认 5 |
| 最大重试次数 | 任务失败后最多重试几次，默认 3 |
| Data（JSON） | 每次触发时传给 Worker 的数据 |

### 快速模板

内置 5 个常用场景模板，点击即可填充表单：

| 模板 | Job Type | 间隔 | 说明 |
|---|---|---|---|
| 💓 心跳检测 | `heartbeat` | 1m | 定期检测服务存活 |
| 📊 生成报告 | `generate_report` | 1h | 定期生成统计报告 |
| 🧹 数据清理 | `cleanup_expired` | 6h | 清理过期数据 |
| 🔄 数据同步 | `data_sync` | 5m | 定期同步外部数据 |
| 🔔 定时通知 | `push_notification` | 30m | 定期推送通知 |

### 任务列表

每个定时任务卡片显示：
- 运行状态（运行中 / 已暂停）
- 执行间隔、队列、Job Type、优先级、重试次数
- 上次触发时间、下次触发倒计时、创建时间

支持操作：
- **▶ 立即触发** — 立即向队列投递一个任务（不影响定时计划）
- **⏸ 暂停 / ▶ 启用** — 切换定时任务的启用状态
- **🗑 删除** — 删除该定时任务

### 实时触发日志

页面每 5 秒轮询一次服务端，自动检测 `last_run_at` 变化，当定时任务被服务端触发时，日志区会实时显示触发记录。

## API 对应关系

| 操作 | HTTP 方法 | 路径 |
|---|---|---|
| 列出所有 cron | GET | `/api/crons` |
| 创建 cron | POST | `/api/crons` |
| 启用/禁用 cron | PATCH | `/api/crons/:id` |
| 删除 cron | DELETE | `/api/crons/:id` |
| 手动触发（投递任务） | POST | `/api/jobs` |

## 执行间隔格式

服务端支持以下格式（由 `parseDuration` 解析）：

| 格式 | 含义 |
|---|---|
| `10s` | 每 10 秒 |
| `5m` | 每 5 分钟 |
| `2h` | 每 2 小时 |
| `1d` | 每 1 天 |
| `1w` | 每 1 周 |

> 注意：目前不支持标准 5 段 cron 表达式（如 `0 3 * * *`），只支持固定间隔。
> 如需精确时间点调度，请参考 [缺失特性说明](../../README.md)。
